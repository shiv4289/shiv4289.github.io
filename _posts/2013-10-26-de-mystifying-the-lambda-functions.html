After grinding through some Ruby codes all day long, I planned to study something light at night and opened up my wishlist to find "Lambda functions" listed there. I have been through Lambda functions many times earlier but I could never understand whats so cool about them for  there are alternate ways of doing things all the time. To have an idea of the same, I googled it out and as it happens most of times, I found a question on stackoverflow which seemed like the exact same question I had in mind. The question titled "Why use lambda functions?"  says:

I can find lots of stuff showing me what a lambda function is, and how the syntax works and what not. But other than the "coolness factor" (I can make a function in middle a call to another function, neat!) I haven't seen something that's overwhelmingly compelling to say why I really need/want to use them. It seems to be more of a stylistic or structural choice in most examples I've seen. How does it make my programs, more correct, more reliable, faster, or easier to understand?

Lets get started with some cool little examples of Lambda functions. Note that I will be mostly using python codes in this post. If you are a C enthusiast (Well me too ;)) you dont need much more than a youtube video to familiarize with the syntax of Lambda functions in C++ 11.

>>> foo = [2, 18, 9, 22, 17, 24, 8, 12, 27]
>>> print filter(lambda x: x % 3 == 0, foo)

[18, 9, 24, 12, 27]

>>> print map(lambda x: x * 2 + 10, foo)

[14, 46, 28, 54, 44, 58, 26, 34, 64]

>>> print reduce(lambda x, y: x + y, foo)

139
